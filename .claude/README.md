# Claude Code — агенты и скилы проекта

## Скилы (skills)

Работают **в контексте текущей беседы** — видят что обсуждалось, какие файлы менялись, какой план. Вызываются через `/имя`.

| Команда | Что делает |
|---------|-----------|
| `/review-bot` | Ревью кода — безопасность, async, Gemini, twitchio паттерны |
| `/refactor-bot` | Рефакторинг с учётом паттернов проекта |
| `/command-bot` | Пошаговое добавление новой `!команды` в бота |
| `/prompt-bot` | Редактирование `prompt.txt` (личность бота) |
| `/db-bot` | Изменения в БД — таблицы, индексы, FTS, запросы |
| `/debug-bot` | Диагностика багов — симптомы → причина → исправление |

### Примеры

```
/review-bot
— проверит последние изменения в контексте того, что обсуждалось

/command-bot
> хочу добавить !top — топ активных пользователей за сессию
— проведёт по чеклисту: триггер, handler, роутинг, help, CLAUDE.md

/prompt-bot
> бот слишком агрессивно отвечает, смягчи тон
— прочитает prompt.txt, предложит изменения

/db-bot
> нужна таблица для хранения реакций на сообщения
— создаст таблицу, индексы, CRUD-функции по паттернам проекта

/debug-bot
> бот не отвечает на !who, хотя раньше работало
— определит область проблемы, проверит код, найдёт причину
```

## Агенты (agents)

Работают **изолированно** — не видят текущую беседу, выполняют задачу с нуля. Вызываются через `@имя`.

| Команда | Модель | Что делает |
|---------|--------|-----------|
| `@architect` | opus | Анализ архитектуры, проектирование фич, планирование рефакторинга |
| `@tester` | sonnet | Написание тестов для модулей |

### Примеры

```
@architect проанализируй модуль database.py и предложи улучшения
@tester напиши тесты для src/utils.py
```

## Когда что использовать

| Ситуация | Что выбрать |
|----------|-------------|
| Только что написал код, хочу проверить | `/review-bot` |
| Обсудили план, нужно рефакторить | `/refactor-bot` |
| Хочу добавить команду в бота | `/command-bot` |
| Нужно поменять поведение бота | `/prompt-bot` |
| Нужна новая таблица или запрос | `/db-bot` |
| Что-то сломалось | `/debug-bot` |
| Нужен глубокий аудит с нуля | `@architect` |
| Нужны тесты для модуля | `@tester` |

**Правило:** если задача зависит от контекста беседы — скил. Если самостоятельная задача с нуля — агент.

## Структура файлов

```
.claude/
├── README.md                          # этот файл
├── agents/
│   ├── architect.md                   # @architect
│   └── tester.md                      # @tester
└── skills/
    ├── review-bot/SKILL.md            # /review-bot
    ├── refactor-bot/SKILL.md          # /refactor-bot
    ├── command-bot/SKILL.md           # /command-bot
    ├── prompt-bot/SKILL.md            # /prompt-bot
    ├── db-bot/SKILL.md               # /db-bot
    └── debug-bot/SKILL.md            # /debug-bot
```

Скилы = директория с `SKILL.md` внутри. Агенты = отдельные `.md` файлы.
