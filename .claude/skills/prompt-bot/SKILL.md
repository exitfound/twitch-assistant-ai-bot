Помоги отредактировать системный промпт бота в `prompt.txt`.

## Как работает prompt.txt

- Файл `prompt.txt` в корне проекта — системная инструкция для Gemini
- Читается при каждом вызове Gemini через `Gemini.get_system_instruction()` → `_load_prompt_sync()`
- Кешируется по mtime — изменения подхватываются без перезапуска бота
- Используется в `make_gen_config()` — стандартные ответы, `!who`, `!versus`, proactive messages
- НЕ используется в `!ask` (свой промпт, без личности) и `!summary` (своя инструкция поверх)

## Что учитывать при редактировании

**Контекст бота:**
- Twitch-чат, русскоязычная аудитория
- Бот отвечает на сообщения с триггером (`сосур*`, `@botname`, реплаи)
- Лимит ответа — 450 символов (Twitch). Промпт должен это учитывать
- Ответы иногда автоматически переводятся в CAPS (30% шанс)
- Бот видит: факты о пользователе, историю чата, базу знаний, FTS-результаты

**Стиль ответов:**
- Plain text, без markdown (markdown стрипается, но лучше не генерировать)
- Краткость — Twitch-чат не для лонгридов
- Уместность — бот виден всем зрителям канала

**Секции контекста которые бот получает:**
```
[Сохранённые факты]
username: факт о пользователе

[Последние сообщения в чате]
username: сообщение

[Контекст канала]
релевантные записи из базы знаний + истории чата

[Язык чата]
случайные записи из базы знаний (язык/стиль канала)

username спрашивает: текст сообщения
```

## Процесс

1. Прочитай текущий `prompt.txt`
2. Обсуди с пользователем что менять
3. Предложи изменения — покажи diff
4. Применив изменение, напомни: промпт подхватится автоматически при следующем вызове Gemini (бот перезапускать не нужно)
